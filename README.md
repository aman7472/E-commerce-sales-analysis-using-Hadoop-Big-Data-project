## 📌 Project Overview

This project presents a complete Big Data pipeline to analyze e-commerce sales data using the Hadoop ecosystem.

With the rapid growth of online shopping, massive transactional datasets are generated daily. This project uses distributed computing to process data efficiently and extract meaningful insights.

---

## 🔍 Key Insights

- 📊 Total Sales & Profit Analysis  
- 🏆 Top-Selling Products  
- 🌍 Region-wise Performance  

---

## 🎯 Objectives

- Store large datasets using **HDFS**  
- Process data using **MapReduce (Python)**  
- Perform analysis using **Hive**  
- Understand distributed data processing  
- Generate business insights  

---

## 🏗️ Architecture

```
Raw CSV Dataset
      ↓
HDFS (Storage)
      ↓
MapReduce (Processing)
      ↓
Processed Data (HDFS)
      ↓
Hive (Analysis)
      ↓
Insights & Reports
```

---

## ⚙️ Tech Stack

- HDFS  
- MapReduce (Python)  
- Hive  
- Linux / Cloudera  
- Generative AI  

---

## 📊 Dataset Details

The dataset contains structured e-commerce transaction data.

### Columns:

- Order Date – Date of purchase  
- Product Name – Product name  
- Category – Product category  
- Region – Sales region  
- Quantity – Units sold  
- Sales – Revenue generated  
- Profit – Profit earned  

---

## 🔄 Workflow

1. Upload dataset to HDFS  
2. Process data using MapReduce  
3. Store processed output in HDFS  
4. Analyze using Hive  
5. Generate insights
